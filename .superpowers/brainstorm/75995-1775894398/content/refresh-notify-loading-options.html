全局体验统一方向

基于当前 Flutter 实现分布，先锁总策略。看哪个更适合这次 worktree 的修复范围。

A. 系统最小改动
Header
刷新圆点仍跟随各页面容器位置
Page content
Toast 只统一颜色，不动交互层

A. 最小修补

风险最低，但刷新位置和 loading 体验仍会残留不一致。

B. 统一交互层
Pinned header
下拉刷新统一从内容起点出现，不压 header
Toast/inline loading/skeleton 走统一组件和语义色

B. 推荐

统一刷新锚点、通知样式和 loading 分层，但不重做每个页面结构。

C. 全量视觉重构
New shell
按信息架构重做 skeleton 和通知表现

C. 大改版

一致性最好，但范围会扩大，不适合这次以修复为主的 worktree。

我目前的判断

B 的收益

  • 能把下拉刷新从“页面各自决定”收敛成共享规则
  • 能统一 toast / success / warning / error 的视觉和行为
  • 能把 full-screen loading、inline loading、layout skeleton 分开治理

B 的代价

  • 需要抽共享容器或约束页面接入方式
  • 骨架屏只会覆盖高频列表页，不会一次性铺满所有页面
