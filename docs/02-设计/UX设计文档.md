### 1.2 导航结构设计

#### 底部 Tab 栏（5个Tab）

```
┌─────────────────────────────────────────────┐
│                                             │
│              [ 页面内容区 ]                  │
│                                             │
├─────────────────────────────────────────────┤
│  🏠首页    📋广场    💬消息    📊项目    👤我的  │
└─────────────────────────────────────────────┘
```

**Tab 说明**：

| Tab | 图标 | 需求方语境 | 供给方语境 |
|-----|------|-----------|-----------|
| 首页 | 房子 | 快速发布需求、查看推荐 | 快速查看推荐需求、收入概览 |
| 广场 | 列表 | 浏览供给方/不常用 | 浏览需求、寻找机会 |
| 消息 | 气泡 | 项目沟通、接收通知 | 项目沟通、投标回复 |
| 项目 | 看板 | 追踪我的项目进度 | 管理我的接单项目 |
| 我的 | 头像 | 个人中心、支付记录 | 个人中心、收入提现 |

#### 页面层级规则

| 层级 | 说明 | 导航方式 |
|------|------|---------|
| L0 | Tab 页（5个主页面） | 底部 Tab 切换，无转场动画 |
| L1 | 二级页面（列表→详情） | 右滑进入，左滑返回 |
| L2 | 三级页面（详情→子详情） | 右滑进入，左滑返回 |
| Modal | 浮层/弹窗（筛选/操作确认） | 底部上滑弹出 |
| Flow | 流程页（发布/支付） | 独立全屏，顶部步骤指示器 |

### 1.3 需求方视角页面流

```
注册登录 → 选择"我是需求方"
    │
    ▼
┌─ 首页 ─────────────────────────────────────────────┐
│  看到："发布需求"大按钮 + 推荐供给方 + 热门分类      │
└────────────────────────────────────────────────────┘
    │ 点击"发布需求"
    ▼
┌─ 需求发布流程 ─────────────────────────────────────┐
│  选类别 → AI对话 → 生成PRD → 确认 → 设置预算       │
└────────────────────────────────────────────────────┘
    │ 发布成功
    ▼
┌─ 等待撮合 ─────────────────────────────────────────┐
│  收到投标通知 → 查看投标列表 → 对比供给方            │
│  → 查看供给方主页/作品 → 选定供给方                  │
└────────────────────────────────────────────────────┘
    │ 确认并支付
    ▼
┌─ 项目进行中 ───────────────────────────────────────┐
│  项目看板（简化视图）→ 查看进度 → IM沟通             │
│  → 接收每日简报 → 查看PRD概览                       │
└────────────────────────────────────────────────────┘
    │ 收到验收请求
    ▼
┌─ 验收流程 ─────────────────────────────────────────┐
│  预览演示 → 逐项确认 → 通过/要求修改                │
│  → 确认付款释放 → 双向评价                          │
└────────────────────────────────────────────────────┘
```

```
│  → 团队内部协作（团队模式）                          │

    │ 阶段交付
    ▼
┌─ 交付验收 ─────────────────────────────────────────┐
│  提交交付物 → 等待验收 → 修改（如有）               │
│  → 验收通过 → 收到付款（团队自动分账）              │
│  → 双向评价 → 作品入库                              │
└────────────────────────────────────────────────────┘
```

### 4.2 页面转场动效建议

| 转场类型 | 动效描述 | 持续时间 | 曲线 |
|---------|---------|---------|------|
| **Push（前进）** | 新页面从右侧滑入，当前页面左移并轻微缩小 | 300ms | ease-out |
| **Pop（返回）** | 当前页面向右滑出，上一页面从左侧还原 | 250ms | ease-in |
| **Modal（弹窗）** | 从底部上滑弹出，背景添加半透明遮罩 | 350ms | spring(0.8) |
| **Tab切换** | 淡入淡出（crossfade），无位移 | 200ms | linear |
| **卡片展开** | 原地展开动画，其他卡片下移让位 | 300ms | ease-out |
| **卡片折叠** | 原地收缩动画，其他卡片上移回位 | 250ms | ease-in |
| **概览→卡片视图** | 树状图缩小淡出，卡片列表淡入上滑 | 400ms | ease-in-out |
| **进度更新** | 进度条/进度环数值平滑过渡 | 600ms | ease-out |
| **状态变更** | 状态色块渐变过渡 + 微弹效果 | 300ms | spring(0.7) |
| **列表项删除** | 向左滑出 + 下方项平滑上移 | 250ms | ease-out |

## 附录 B：设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 底部导航Tab数量 | 5个 | 涵盖核心场景，不超过iOS人机规范推荐上限 |
| 首页差异化 | 需求方/供给方不同首页 | 两类用户核心诉求完全不同，差异化首页提升转化 |
| PRD展示 | 概览+卡片双视图 | 降低认知压力，渐进式信息披露 |
| EARS卡片默认态 | 折叠 | 避免信息过载，用户按需展开 |
| 主色选择 | 活力蓝 #0A7AFF | 专业+信任+科技感，适合双端 |
| 组队入口位置 | 首页提示条+独立页面 | AI主动提示降低发现成本，独立页面满足主动寻找 |
| 角色切换方式 | "我的"页面内切换 | 低频操作，不占据主导航位 |
| 看板交互 | 三列看板+长按拖拽 | 符合项目管理直觉，降低学习成本 |
