## 一、我们是谁

我们连接两类人：**有想法、有需求的项目方**，以及**掌握 AI 能力的团队方**——无论是 Vibe Coding 开发、数据服务，还是 AI Agent Skill，开造都能帮你 AI 拆需求、智能匹配、担保交付。

**开造用三大 AI Agent（需求分析 Agent、项目管理 Agent、质量检查 Agent）贯穿全流程**，从根本上解决供需两端的信息不对称和信任缺失。

## 二、平台六大核心能力

### 1. 智能撮合引擎

开造的撮合不是简单的信息发布和浏览，而是基于 AI 的多维度精准匹配。

**五维加权匹配算法：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 技能匹配度 | 30% | 团队方技能标签与项目技术需求的语义匹配 |
| 历史评价 | 25% | 综合评分、按时交付率、好评率 |
| 价格匹配 | 20% | 团队方报价与项目方预算的吻合度 |
| 响应速度 | 15% | 平均响应时间和在线活跃度 |
| 作品相似度 | 10% | 历史作品与当前需求的向量相似度 |

**三种撮合模式：**

- **主动投标**：团队方浏览项目广场，主动提交方案和报价
- **智能推荐**：AI 根据匹配度主动向双方推送最合适的对象
- **快速匹配**：项目方选择「急速匹配」，系统自动分配匹配度最高的团队

项目发布后，平台实时推荐 3-5 个高匹配度的团队方供选择，同时开放投标通道让更多团队主动参与竞争，确保项目方总能找到最合适的合作方。

**为什么不怕跳单？** 开造做的不是"介绍认识"这一个动作，而是全流程价值嵌入——AI 项目管理工具、资金托管、争议仲裁、信用累积、AI 质检、续单优惠，每个环节都依赖平台工具。用户不是为"认识"付佣金，是为整套服务付费。在平台交易比私联更划算，跳单反而亏了。

### 2. AI 需求自动化对齐

**这是开造最核心的差异化能力。**

传统外包行业最大的痛点是「需求对不齐」——项目方说不清楚想要什么，团队方理解有偏差，导致反复返工和纠纷。开造用 AI 从根本上解决了这个问题。

**工作流程：**

```
项目方用大白话描述想法
    ↓
AI 多轮对话（7-10 轮），围绕 7 大维度智能引导澄清
    ↓
自动生成结构化项目需求文档（PRD）
    ↓
项目方确认，允许补充修改
    ↓
撮合成功后，平台介入对齐需求，确认合作
    ↓
AI 进一步拆解为 EARS 原子级任务卡片 + 生成架构设计文档 + 项目管理方案
    ↓
双方基于同一份标准文档开始协作
```

**AI 澄清的 7 大维度：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 产品定位 | 20% | 做什么、不做什么 |
| 用户画像 | 15% | 谁用、什么场景下用 |
| 核心功能 | 20% | 功能模块和关键交互 |
| 技术偏好 | 10% | 平台、技术栈要求 |
| 商业目标 | 10% | 变现方式和商业模式 |
| MVP 范围 | 15% | 第一版优先做哪些 |
| 约束条件 | 10% | 预算、工期、特殊要求 |

所有维度信息充分度 ≥ 60 且总分 ≥ 80 时，AI 自动触发 PRD 生成。项目方不需要懂技术术语，团队方拿到的是清晰、无歧义、可验收的标准文档——双方从第一天起就对齐了预期。

**EARS 任务卡片体系：** 将模糊需求转化为原子级、无歧义、可验收的任务单元，是行业内首创的 AI 需求工程方法论在撮合平台的应用。每张卡片包含需求描述、验收标准、预估工时、依赖关系，让交付质量有据可查。

### 3. 项目管理与协作

撮合只是开始，开造在项目执行阶段同样提供全程 AI 管理支持。

**项目管理看板（撮合成功后开启）：**

| 功能 | 说明 |
|------|------|
| 任务看板 | 待办 / 进行中 / 已完成，三列看板直观展示，团队方拖拽操作 |
| 里程碑管理 | 时间线展示，含支付比例和关键节点 |
| PRD 在线查看 | 需求文档随时回溯 |
| 文件协作 | 项目文件双方共享上传下载 |
| AI 进度管理 | 延期风险预警、阻塞点识别、每日简报、智能催促 |

**双视图设计（降低认知压力）：**

- **项目方看概览**：模块级进度，绿色=完成，蓝色=进行中，灰色=待开始，一屏掌握全貌
- **团队方看卡片**：EARS 级原子任务，按状态/角色/优先级筛选，只看与自己相关的部分

信息渐进呈现——默认只展示 5-8 个模块级概览卡片，已完成的自动折叠，当前里程碑高亮，未来里程碑置灰，引导用户专注当下。

### 4. 支付管理与交易保障（暂未提供，后期上线）

开造提供全流程资金保障，让双方交易安全无忧。

**担保交易机制：**

```
项目方预付款 → 资金进入平台担保（微信支付/支付宝电商收付通）
    ↓
团队方按里程碑交付 → 项目方验收通过
    ↓
资金按阶段释放给团队方
```

**核心特点：**

- **分阶段付款**：按里程碑分批支付（如 30% + 40% + 30%），降低单次风险
- **不满意不付款**：验收通过才释放资金，项目方权益有保障
- **合规安全**：全程接入持牌第三方支付，资金不进平台账户，不碰资金沉淀，符合国家监管要求
- **自动对账**：交易明细、账单全流程线上化

**数据服务 Token 计价：** 数据商接入后，数据查询、清洗、标注、增强等服务采用 Token 计价模式——类似大模型 API 的计费逻辑，按实际消耗计费，按需付费、成本可预测。

### 6. 沙箱验证与质量保障

开造为每个项目提供沙箱验证环境和 AI 质量检查体系，帮助项目方在验收前全面评估交付物质量。

**沙箱验证环境：**

- 每个项目自动创建隔离的沙箱环境，供团队开发、测试和演示
- 项目方可直接在沙箱中体验和验收，不怕"本地能跑线上不行"
- 支持一键切换不同大模型（通义千问/DeepSeek/Claude），根据任务复杂度自动推荐最优模型

**AI 质量检查能力：**

| 检查维度 | 说明 |
|---------|------|
| 功能完整性 | 对照 PRD 和 EARS 卡片验收标准，逐项检查功能是否实现 |
| 代码基本规范 | 代码结构、命名规范、安全基线自动扫描（Semgrep） |
| 响应式适配 | 移动端/桌面端页面适配检查（Playwright） |
| 性能基线 | 页面加载速度、基础性能指标评估（Lighthouse） |

**验收流程：**

```
团队方提交阶段交付物
    ↓
AI 质量检查 Agent 自动预检（生成检查报告）
    ↓
项目方在沙箱中在线预览/体验
    ↓
通过 → 释放该阶段资金
需修改 → 附修改意见，团队方迭代后重新提交
```

每张 EARS 任务卡片都有明确的验收标准，AI 对照标准自动检查，大幅降低交付争议。项目方不需要懂技术，看检查报告就能判断交付质量。

## 三、团队方评级体系

开造建立了专业的团队方评级体系（**vc-T 体系**），帮助项目方快速识别团队能力。

**五维评估模型（满分 750 分）：**

| 维度 | 权重 | 满分 |
|------|------|------|
| 技术深度 | 30% | 225 |
| 项目经验 | 25% | 187.5 |
| AI 工具熟练度 | 20% | 150 |
| 作品集质量 | 15% | 112.5 |
| 教育背景 | 10% | 75 |

**十级等级序列：** vc-T1（初启）→ vc-T2（胜任）→ vc-T3（熟练）→ vc-T4（出众）→ vc-T5（资深）→ vc-T6（卓越）→ vc-T7（杰出）→ vc-T8（非凡）→ vc-T9（登峰）→ vc-T10（传奇），首次定级上限 vc-T5

等级根据平台内的交付表现动态调整：

| 行为 | 积分变动 |
|------|---------|
| 完成项目 | +30 ~ +100 |
| 五星好评 | +20 |
| 准时交付 | +15 |
| 提前交付 | +25 |
| 复购客户 | +30 |
| 逾期 | -20 |
| 差评 | -30 |
| 中途退出 | -50 |

真实反映团队的持续表现，让优秀的团队脱颖而出。
