## 1. 小红书品牌定位

### 1.1 账号人设定义

| 维度 | 内容 |
|------|------|
| **账号名称** | 开造开造（主号）/ 开造·创造营（内容号） |
| **头像风格** | 渐变色系（紫→蓝科技感）+ 简洁「VB」字母Logo，风格参考 Notion/Linear 的极简科技美学 |
| **简介文案** | 「让每个好想法都能被低成本实现 💡 Vibe Coding 撮合平台 · 连接需求方与一人公司 · 1/5成本 1/10时间交付软件产品 · 入驻接单/发布需求👇」 |
| **人设定位** | "懂技术的创业搭子"——不是冷冰冰的平台官方号，而是一个懂 AI 编程、懂创业者痛点、愿意分享干货的"创业老友" |
| **说话风格** | 专业但不晦涩，用大白话讲技术趋势；多用数据说话；偶尔自嘲和玩梗；善于用对比制造冲击感（"以前花3万做个APP，现在3天搞定"） |

### 1.2 内容调性和视觉风格

**内容调性**：
- **主基调**：实用主义 + 未来感——"AI 时代的新机会，普通人也能抓住"
- **情绪价值**：激发行动力（"你也可以"）而非制造焦虑
- **知识密度**：每条内容至少包含一个可执行的信息点
- **互动感**：多提问、多做选择题（"你觉得哪个方案更好？"），引导评论

**视觉风格规范**：

| 元素 | 规范 |
|------|------|
| **主色调** | #6C5CE7（科技紫）+ #0984E3（信任蓝）+ #FFFFFF（留白） |
| **辅助色** | #00B894（成功绿）用于数据亮点、#FD79A8（活力粉）用于强调 |
| **字体** | 标题用思源黑体 Bold，正文用苹方 Regular |
| **封面模板** | 统一左上角带「开造」水印Logo；大标题居中，字号突出；底部统一信息条 |
| **图文排版** | 单图信息密度适中，轮播图每页一个核心信息点，配合图标/插画辅助理解 |
| **视频风格** | 屏幕录制 + 真人口播混剪；节奏快（3秒内进入主题）；字幕常驻 |

### 1.3 核心关键词策略（小红书 SEO）

**一级关键词（高搜索量，核心卡位）**：
- AI编程、氛围编程、vibe coding、AI开发、AI写代码

**二级关键词（精准长尾，转化导向）**：
- AI编程接单、零代码赚钱、一人公司、独立开发者赚钱、AI做APP
- 低成本创业、副业赚钱、AI工具推荐、程序员副业

**三级关键词（场景词，覆盖搜索意图）**：
- 小程序开发多少钱、做个APP要多少钱、找人做网站、外包开发
- Cursor教程、Lovable教程、Bolt教程、Claude编程

**关键词布局策略**：
- 标题必含1个一级词 + 1个二级词
- 正文前3行自然植入2-3个关键词
- 标签区固定放置5-8个混合层级关键词
- 每周根据小红书搜索热度调整关键词优先级

### 1.4 竞品账号分析

| 账号类型 | 代表账号 | 粉丝量级 | 内容特点 | 我们的差异化 |
|----------|---------|---------|---------|------------|
| **AI编程教程类** | "AI编程小助手"、"Cursor玩家" | 1-5万 | 工具教程为主，偏技术向 | 我们聚焦"变现+接单"场景，不只是教工具 |
| **独立开发者类** | "一个人的产品"、"独立开发日记" | 2-10万 | 个人开发故事、产品分享 | 我们提供撮合平台，从故事到行动的闭环 |
| **副业赚钱类** | "程序员副业指南"、"AI赚钱术" | 5-20万 | 泛副业内容，AI编程只是其中一个话题 | 我们垂直深耕 vibe coding 赛道，更专业 |
| **外包接单类** | "接单程序员"、"自由开发者联盟" | 1-3万 | 传统外包经验分享 | 我们强调 AI 时代的新模式，效率碾压传统外包 |

**竞品内容缺口（我们的机会）**：
1. 缺少"需求方视角"内容——大多数账号只服务供给方（开发者），忽略了有需求的创业者
2. 缺少"交易保障"话题——用户担心被骗、交付质量差，但没有账号系统性地讲这件事
3. 缺少"组队协作"内容——vibe coding 的协作模式几乎无人覆盖
4. 缺少"真实案例+数据"——多数内容停留在概念层面，缺少真实交付案例和收入数据
