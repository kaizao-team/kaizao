# 开造 Flutter 前端设计规范

> 风格定位：**Quiet Craft** — 安静、精致、有温度。
> 参考调性：Notion 的克制 × Luma 的透气 × ChatGPT 的现代感。
> 核心原则：**不用渐变证明高级，用留白和层次说话。**

---

## 1. 色彩系统

### 1.1 品牌色

| Token | Hex | 用途 |
|-------|-----|------|
| `primary` | `#7C3AED` | 选中态、链接文字、小面积强调。**绝不做大面积填充** |
| `primaryLight` | `#F3EEFF` | 标签底色、图标容器底色、hover 态 |
| `secondary` | `#3B82F6` | 信息提示、次级链接（几乎不用） |

### 1.2 中性色

| Token | Hex | 用途 |
|-------|-----|------|
| `background` | `#F6F6F6` | 全局画布。暖白，不是冷灰 |
| `surface` | `#FFFFFF` | 卡片、浮层、输入框内部 |
| `surfaceContainer` | `#F0F1F1` | 输入框底色、分组背景、次级面板 |
| `border` | `#E5E5E5` | 仅在输入框 focus 时或分隔线场景使用。**默认不用边框** |

### 1.3 文字色

| Token | Hex | 用途 |
|-------|-----|------|
| `textPrimary` | `#1A1A1A` | 标题、按钮文字。**不用纯黑 #000** |
| `textSecondary` | `#6B7280` | 正文、描述 |
| `textTertiary` | `#9CA3AF` | 占位符、时间戳、辅助信息 |

### 1.4 语义色

| Token | Hex | 用途 |
|-------|-----|------|
| `success` | `#22C55E` | 完成、通过、在线 |
| `warning` | `#F59E0B` | 评分星星、提醒 |
| `error` | `#EF4444` | 错误、逾期、退出 |

### 1.5 渐变规则

**全局禁止渐变**，以下两处例外：

1. **Splash Screen 背景**：`#1E1B4B → #0F172A`（深空渐变，仅此一处）
2. **Logo 三角形绘制**：`#7C3AED → #6366F1 → #3B82F6`（仅 logo 本身）

其他一切 UI 元素（按钮、卡片、进度条、标签、背景）一律使用纯色。

## 2. 排版系统

### 2.1 字体

- 英文标题：**Plus Jakarta Sans**（如可用），否则系统默认
- 中文 & 正文：**系统默认**（iOS: PingFang SC / Android: Noto Sans CJK）
- 数字/数据：**DIN / SF Pro**（如可用），否则系统默认

### 2.2 字阶

| Token | Size | Weight | Line Height | 用途 |
|-------|------|--------|-------------|------|
| `displayLg` | 28px | Bold (700) | 1.3 | 页面大标题（极少用） |
| `headlineMd` | 22px | Bold (700) | 1.3 | 区块标题 |
| `titleLg` | 18px | SemiBold (600) | 1.4 | 卡片标题 |
| `titleMd` | 16px | SemiBold (600) | 1.4 | 列表项标题 |
| `bodyLg` | 16px | Regular (400) | 1.6 | 正文 |
| `bodyMd` | 14px | Regular (400) | 1.6 | 次级正文 |
| `labelLg` | 13px | Medium (500) | 1.3 | 按钮文字、标签 |
| `labelMd` | 12px | Medium (500) | 1.3 | 辅助标签、Badge |
| `caption` | 11px | Regular (400) | 1.3 | 时间戳、脚注 |

### 2.3 规则

- 中文 body 的 `lineHeight` ≥ 1.6，避免"密块感"
- 标题使用 `letterSpacing: -0.02em`（微收紧，更精致）
- 每个页面最多 3 层文字层级（标题 / 正文 / 辅助）
- 英文品牌名"Kaizao"始终用 `w700`，不加引号

## 3. 间距系统

### 3.1 基础间距（4px 基准）

| Token | Value |
|-------|-------|
| `xs` | 4px |
| `sm` | 8px |
| `md` | 12px |
| `lg` | 16px |
| `xl` | 20px |
| `xxl` | 24px |
| `xxxl` | 32px |

### 3.2 屏幕边距

- 左右安全边距：**20px**（贯穿全 App）
- 底部导航栏上方留白：根据 SafeArea 自适应

### 3.3 区块间距

- 卡片与卡片之间：16px
- 区块标题与内容之间：16px
- 区块与区块之间：32px
- 卡片内部 padding：16–20px

---

## 4. 圆角系统

| Token | Value | 用途 |
|-------|-------|------|
| `radiusSm` | 8px | 小标签、Badge |
| `radiusMd` | 12px | 按钮、输入框、消息气泡 |
| `radiusLg` | 16px | 卡片、浮层 |
| `radiusXl` | 24px | 底部弹窗 sheet |

### 圆角规则

- **禁止使用正圆形容器**（`borderRadius: 50%`）做图标/分类入口。使用 squircle（连续曲率圆角方形），`borderRadius: 14px`
- **头像是唯一例外**：用户头像可以用 `CircleAvatar`
- 按钮绝不做 pill 形（`borderRadius: 999`），统一 12px

---

## 5. 阴影系统

### "无线分隔"原则

**禁止使用 1px 分隔线**区分内容区域。通过以下方式替代：

1. **色调层叠**：卡片（#FFFFFF）放在画布（#F6F6F6）上，亮度差天然形成"浮起"
2. **间距呼吸**：区块间用 32px 间距替代分隔线
3. **微阴影**：仅在卡片需要明确"浮起"时使用

### 阴影参数

| Level | 场景 | CSS |
|-------|------|-----|
| `elevation0` | 普通卡片 | 无阴影，仅靠色差 |
| `elevation1` | 可交互卡片 | `0 2px 8px rgba(0,0,0,0.04)` |
| `elevation2` | 浮层/弹窗 | `0 8px 24px rgba(0,0,0,0.08)` |

## 6. 组件规范

### 6.1 按钮

| 类型 | 背景 | 文字 | 圆角 | 场景 |
|------|------|------|------|------|
| Primary | `#1A1A1A`（深灰黑） | White | 12px | 主 CTA，每页最多 1 个 |
| Secondary | `#F0F1F1` | `#1A1A1A` | 12px | 次级操作 |
| Ghost | 透明 | `#7C3AED` | — | 文字链接、"查看全部" |
| Danger | 透明 + border `#EF4444` | `#EF4444` | 12px | 退出登录等 |

**注意**：Primary 按钮背景是**深灰黑**，不是紫色。紫色仅用于文字链接和小面积强调。

### 6.2 卡片

- 背景：`#FFFFFF`
- 圆角：16px
- 内边距：16–20px
- 阴影：elevation1（可选，也可只靠色差）
- **无边框**。如果视觉上确实需要边界，用 `#F6F6F6` 背景对比
- 卡片之间间距：12–16px

### 6.3 输入框

- 背景：`#F0F1F1`
- 圆角：12px
- 高度：48px
- 无边框。Focus 态可加 2px bottom accent `#7C3AED`
- placeholder 色：`#9CA3AF`

### 6.4 分类图标容器

- 尺寸：56×56px
- 背景：`#F3EEFF`（primaryLight）
- 圆角：14px（squircle）
- 图标尺寸：24px
- 图标色：`#7C3AED`

### 6.5 Badge / Tag

- 背景：对应语义色的 10% 透明度
- 文字：对应语义色
- 圆角：6px
- padding：水平 8px，垂直 3px
- 字号：11px，Medium

### 6.6 底部导航栏

- 背景：`#FFFFFF`
- 上边缘：0.5px `rgba(0,0,0,0.05)` 线（唯一允许的"线"）
- 5 个 Tab：首页 / 广场 / 消息 / 项目 / 我
- 选中色：`#7C3AED`（实心图标）
- 未选中色：`#9CA3AF`（线条图标）
- 标签字号：11px

### 6.7 列表项

- 无分隔线。靠上下 padding（12–16px）呼吸
- 右侧箭头：`chevron_right`，20px，`#9CA3AF`
- 左侧图标容器：36×36px squircle（10px 圆角），语义色 10% 透明底

## 7. 动效规范

| 参数 | 值 |
|------|-----|
| 标准时长 | 200–300ms |
| 缓动曲线 | `Curves.easeOutCubic`（Flutter）/ `cubic-bezier(0.16, 1, 0.3, 1)` |
| 页面转场 | 右滑入，MaterialPageRoute 默认 |
| 最小可点击区域 | 44×44px |

### 动效原则

- **克制**。动效是为了给用户反馈，不是为了炫技
- 页面滚动：自然物理滚动，不加额外弹性
- 卡片点击：无缩放动画（避免"果冻感"）。仅用 InkWell 水波纹
- Splash → 首页：淡入动画，2s 后自动跳转

## 8. 关于"AI 感"的约束

以下元素被**明确禁止**，因为它们会让 App 看起来像"又一个 AI 工具"：

- ❌ 大面积紫色/蓝色渐变背景
- ❌ 发光（glow）/ 光晕（halo）效果
- ❌ 毛玻璃（glassmorphism）装饰
- ❌ 流动的 blob 装饰（除 Splash Screen 外）
- ❌ 彩虹色 / 全息色（holographic）
- ❌ 闪烁 / 呼吸灯动画
- ❌ 圆形图标容器

**允许的"AI 暗示"**：
- ✅ `auto_awesome` 图标（✨ 星芒图标）出现在 AI 相关入口
- ✅ AI 生成的内容用浅蓝底（`#EFF6FF`）+ 蓝色文字标注
- ✅ 步骤进度条用朴素的线性样式
