# 需求端与专家端 — 功能模块对照与 Gap 分析

> 文档版本：v1.0
> 日期：2026-03-24
> 依赖文档：PRD v1.0、前端交互 EARS 需求卡片 v1.1、当前代码库
> 目的：梳理两端在每个功能模块中应展示的内容差异，标注当前代码实现与设计文档之间的 Gap，并给出实现优先级建议。

---

## 1. 角色体系说明

| 角色 | 代码标识 (`userRole`) | 说明 |
|------|----------------------|------|
| 需求方（发起人） | `1` | 有想法和需求，找专家帮忙实现 |
| 专家（造物者） | `2` | 擅长 Vibe Coding，接项目交付 |

角色在注册引导流程中选择（`role_select_page.dart`），存储在 `AuthState.userRole` 中，可通过"我的"页面的"切换角色"功能切换。

---

## 2. 底部 Tab 栏总览

5 个 Tab 两端共享，但**内容需要角色化展示**：

| Tab | 路由 | 需求方视角 | 专家视角 | 角色差异程度 |
|-----|------|-----------|---------|------------|
| 首页 | `/home` | AI 对话入口 + 热门分类 + 我的项目 + 推荐专家 | 收入概览 + 推荐需求 + 技能热度 + 组队机会(v2) | **高** — 完全不同的布局 |
| 广场 | `/square` | 浏览模式（发现专家、查看案例） | 投标模式（浏览需求、匹配度排序、AI 组队建议） | **中** — 卡片内容和操作不同 |
| 消息 | `/chat` | 与专家沟通 | 与需求方沟通 | **低** — 共享，无结构差异 |
| 项目 | `/projects` | 我发布的需求列表 + 进度追踪 + 验收入口 | 我参与的项目列表 + 任务看板 + 提交交付 | **高** — 列表来源和操作完全不同 |
| 我的 | `/profile` | 基本信息 + 发布历史 + 消费数据 + 切换角色 | 技能标签 + 作品集 + 接单数据 + 收入/提现 + 切换角色 | **高** — 展示维度不同 |

## 3. 各模块角色差异详细对照

### 3.1 首页 (`HOME`)

| 区块 | 需求方 | 专家 |
|------|--------|------|
| 顶部卡片 | AI 对话入口（"描述你的需求…"），点击进入需求发布 | 收入概览卡片（本月收入、趋势、"查看"按钮 → 钱包页） |
| 第二区块 | 热门分类 Grid（6 类），点击进入带预选分类的需求发布 | 推荐需求列表（AI 匹配的需求卡片，含匹配度标签） |
| 第三区块 | 我的项目（最近项目卡片，"查看全部"→ 项目列表） | 技能热度排行（当前热门技能标签） |
| 第四区块 | 推荐供给方（横向滚动专家卡片） | 组队机会（v2，AI 推荐的组队项目） |

**EARS 卡片参考**：`FE-HOME-001`~`FE-HOME-007`

### 3.2 广场 (`MRKT`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 页面标题 | 需求广场（或"找专家"） | 需求广场 |
| 列表内容 | 浏览已发布的需求（查看进展） / 浏览专家主页 | 浏览所有需求，按匹配度排序 |
| 卡片差异 | 无匹配度标签，无 AI 组队建议 | 显示匹配度渐变标签、AI 组队建议条 |
| 排序选项 | 最新 / 预算 | 最新 / 预算 / **匹配度**（专家专属） |
| 操作入口 | 查看详情 → 可查看投标列表 | 查看详情 → 可投标 |
| 筛选 | 分类、预算范围 | 分类、预算范围、技能匹配 |

**EARS 卡片参考**：`FE-MRKT-001`~`FE-MRKT-007`

### 3.3 需求发布 (`POST`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 入口 | 首页 AI 卡片 / 热门分类 / 底栏"+"（规划中） | **无入口**（专家不发布需求） |
| 流程 | Step1 分类 → Step2 AI 对话 → Step3 PRD 预览 → Step4 预算 → Step5 撮合模式 | N/A |

**EARS 卡片参考**：`FE-POST-001`~`FE-POST-010`

### 3.4 PRD/EARS 视图 (`PRD`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 可见 Tab | 仅"概览视图" | "概览视图" + "卡片视图" |
| 概览视图 | 模块树状图（功能名 + 完成状态 + 进度） | 模块树状图（功能名 + 完成状态 + 预估工时） |
| 卡片视图 | 不可见 | EARS 卡片列表，支持按角色/状态/优先级筛选 |
| 信息隐藏 | 隐藏 EARS 类型标签、技术角色标签、依赖关系、预估工时 | 全部可见 |
| 操作权限 | 只读 | 可勾选验收标准、更新任务状态 |

**EARS 卡片参考**：`FE-PRD-001`~`FE-PRD-009`

### 3.5 智能撮合/投标 (`MTCH`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 投标列表 | 查看收到的投标（比较报价、选定供给方） | 不可见（只看自己的投标状态） |
| 投标表单 | 无 | 填写报价、工期、方案，可选个人/团队投标 |
| AI 推荐 | 查看 AI 推荐的供给方排名 | 查看 AI 对项目的报价建议 |
| 核心操作 | "选 TA" 按钮 → 确认 → 支付 | "提交投标" 按钮 |

**EARS 卡片参考**：`FE-MTCH-001`~`FE-MTCH-005`

### 3.6 项目管理 (`PROJ`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 项目列表 | "我发布的需求"，按状态分组 | "我参与的项目"，按状态分组 |
| 项目详情 | 查看进度、里程碑、AI 简报 | 查看+操作任务看板、提交阶段性成果 |
| 看板操作 | 只读（查看任务状态） | 可拖拽任务卡片、标记完成 |
| 里程碑 | 查看 + 触发验收 | 查看 + 提交交付物 |
| AI 简报 | 查看每日简报 | 查看每日简报 |
| 底部操作 | "沟通"、"查看进度"、触发验收 | "沟通"、"投标"、提交任务 |

**EARS 卡片参考**：`FE-PROJ-001`~`FE-PROJ-008`

### 3.7 验收流程 (`ACCP`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 角色 | 验收方（主导） | 被验收方 |
| 操作 | 逐项勾选验收清单、预览演示、通过/要求修改、释放付款 | 查看验收状态、收到修改请求、重新提交 |

**EARS 卡片参考**：`FE-ACCP-001`~`FE-ACCP-004`

### 3.8 支付结算 (`PAY`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 入口 | 选定供给方后进入支付 / 验收通过后释放付款 | 钱包页查看收入和提现 |
| 支付 | 唤起微信/支付宝支付 | 无 |
| 提现 | 无 | 提现到微信/支付宝/银行卡 |
| 优惠券 | 选择和使用 | 无 |

**EARS 卡片参考**：`FE-PAY-001`~`FE-PAY-005`、`FE-PROF-005`

### 3.9 个人主页 (`PROF`)

| 维度 | 需求方 | 专家 |
|------|--------|------|
| 头部 | 头像 + 昵称 + 简介 | 品牌渐变背景 + 头像 + 昵称 + 等级徽章 |
| 数据统计 | 发布数、消费金额、在平台天数 | 完成项目数、好评率、收入、信用分 |
| 技能标签 | 无 | 技能标签 Wrap + 编辑入口 |
| 作品集 | 无 | 作品图片 Grid + 全屏预览 |
| 操作项 | 切换角色、设置 | 切换角色、钱包/提现、设置 |

**EARS 卡片参考**：`FE-PROF-001`~`FE-PROF-005`

### 3.10 消息 (`CHAT`)

消息模块两端**共享**，无结构差异。双方都可以：
- 查看会话列表
- 发送/接收消息
- 查看关联任务卡片
- 左滑标记已读/删除

**EARS 卡片参考**：`FE-CHAT-001`~`FE-CHAT-006`

### 3.11 引导流程 (`ONBOARD`)

| 需求方引导 (4 步) | 专家引导 (3 步) |
|------------------|----------------|
| 1. 编辑基础资料（昵称） | 1. 编辑专家资料（昵称+技能+工具+自评+排期+预算） |
| 2. 引导点击"+"创建需求 | 2. 引导补充评估信息（作品集+简介） |
| 3. 引导填写需求和预算 | 3. AI 定级展示（等级徽章揭晓） |
| 4. 引导发起沟通 | — |

**EARS 卡片参考**：`FE-ONBOARD-001`~`FE-ONBOARD-009`

## 4. 代码实现 Gap 清单

### 4.1 已实现的角色分流

| 模块 | 实现状态 | 说明 |
|------|---------|------|
| 角色选择 | ✅ 已实现 | `role_select_page.dart` 提供 需求方(1) / 专家(2) 选择 |
| 角色存储 | ✅ 已实现 | `AuthState.userRole` + `StorageService` 持久化 |
| 角色切换 | ✅ 已实现 | `RoleSwitchDialog` + `authStateProvider.selectRole()` |
| 引导流程分叉 | ✅ 已实现 | 需求方走 `ONBOARD-001~004`，专家走 `ONBOARD-005~007` |
| 首页 Provider | ✅ 已实现 | `HomeNotifier` 根据 `_isDemander` 调用不同 API |
| PRD 角色视图 | ✅ 已实现 | `prd_page.dart` 根据 `isDemander` 控制信息展示 |

| G-07 | **个人主页角色化** | `profile_page.dart` 目前两端展示相同内容。专家应额外展示技能标签编辑、作品集、收入统计。需求方应展示发布历史、消费统计。 | `profile_page.dart` | **P1** |
| G-08 | **广场 Tab 语义** | 需求方视角的广场应偏向"找专家"（浏览专家主页和案例），当前仅是需求列表。可能需要 Tab 或视图切换。 | `market_page.dart` | **P2** |
| G-09 | **首页假数据** | 首页推荐供给方（5 个"阿杰"）、我的项目（2 个固定卡片）为硬编码静态数据，未接入 `homeStateProvider`。 | `home_page.dart` | **P0** |
| G-10 | **项目列表假数据** | `project_list_page.dart` 使用本地 `_projects` Map 列表，未接入任何 provider/repository。 | `project_list_page.dart` | **P0** |

## 5. 实现优先级建议

### P0 — 必须立即做（影响核心体验）

1. **G-01 + G-02：首页角色分流**
   - `home_page.dart` 根据 `authState.userRole` 切换需求方视图 / 专家视图
   - 对接已有的 `homeStateProvider`（Provider 已支持角色分流）
   - 引用已有的 `home_expert_section.dart`、`expert_home_demands.dart`、`expert_home_revenue.dart`

2. **G-09 + G-10：首页和项目列表对接真实数据**
   - 首页替换硬编码卡片，使用 `homeStateProvider` 数据
   - 项目列表替换本地 `_projects`，使用 `project_repository` 的真实 API

3. **G-04：项目列表角色化**
   - 需求方请求"我发布的需求"
   - 专家请求"我参与的项目"

### P1 — 尽快实现（完善角色体验）

4. **G-03：广场页面角色化**
   - 专家视角添加匹配度标签
   - 排序选项根据角色动态调整

5. **G-05 + G-06：项目详情/管理操作权限**
   - 需求方看到"验收"入口
   - 专家看到"提交交付"入口
   - 需求方看板为只读

6. **G-07：个人主页角色化**
   - 专家展示技能标签、作品集、接单统计
   - 需求方展示发布历史、消费数据

### P2 — 后续迭代

7. **G-08：广场语义重定义**
   - 需求方的广场偏向"找专家"
   - 考虑加入专家列表 / 案例展示子 Tab

## 6. 模块架构总览

公共模块

├── HOME — 首页需求方视图（AI 入口 + 热门分类 + 我的项目 + 推荐专家）
├── POST — 需求发布（仅需求方）
├── MRKT — 广场浏览模式
├── PRD — PRD 概览视图（简化，隐藏技术细节）✅ 已实现
├── MTCH — 查看投标列表，选定供给方
├── PROJ — 我发布的需求（进度追踪 + 触发验收）
├── ACCP — 验收流程（验收方）
├── PAY — 支付结算（付款方）

│
专家端（专家视角）
├── HOME — 首页专家视图（收入概览 + 推荐需求 + 技能热度）

├── PRD — PRD 完整视图（EARS 卡片 + 任务操作）✅ 已实现
├── MTCH — 投标表单（报价 + 工期 + 方案）
├── PROJ — 我参与的项目（任务看板 + 提交交付）

├── PROF — 个人主页（技能 + 作品集 + 接单数据 + 收入）
