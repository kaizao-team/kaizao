# 角色分流接口变更记录

> 本文档记录角色分流功能实现过程中所有新增/调整的 API 接口及 Mock 数据。

---

## 第 1 期

### 1. 首页聚合接口（已有，无变更）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/home/demander` | GET | 需求方首页数据（AI提示、热门分类、我的项目、推荐专家） |
| `/api/v1/home/expert` | GET | 专家首页数据（收入概览、推荐需求、技能热度、组队机会） |

Mock 文件：`lib/core/mock/mock_data/home_mock.dart`

### 2. 项目列表接口（已调整）

| 接口 | 方法 | 变更说明 |
|------|------|---------|
| `/api/v1/projects` | GET | 新增 `role` query 参数：`1`=需求方（返回我发布的需求），`2`=专家（返回我参与的项目） |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| role | int | 否 | 用户角色：1=需求方，2=专家。默认 1 |

**响应结构不变，data 为 ProjectModel 数组。**

Mock 文件：`lib/core/mock/mock_data/project_mock.dart`

---

## 第 2 期

### 3. 个人资料统计字段扩展

| 接口 | 方法 | 变更说明 |
|------|------|---------|
| `/api/v1/users/:id` | GET | `stats` 对象新增需求方字段 |
| `/api/v1/users/me` | GET | 同上 |

**新增 stats 字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `published_projects` | int | 需求方已发布需求数 |
| `total_spent` | double | 需求方累计消费金额 |
| `days_on_platform` | int | 在平台天数 |

Mock 文件：`lib/core/mock/mock_data/profile_mock.dart`

### 4. 广场排序选项角色化（前端逻辑，无接口变更）

需求方排序选项不展示"匹配度"，通过前端 `MarketSortOption.forRole(role)` 控制，不涉及后端接口。

### 5. 项目详情底部操作（前端路由，无接口变更）

按角色展示不同按钮和路由跳转，不涉及后端接口。

### 6. 看板只读模式（前端 UI，无接口变更）

需求方进入看板时传 `readOnly: true`，禁用拖拽和占位虚线框，不涉及后端接口。

---

## 第 3 期

### 7. 专家列表接口（新增）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/market/experts` | GET | 获取平台专家列表（需求方"找专家"Tab 使用） |

**响应结构：**

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "id": "exp_01",
      "nickname": "阿杰",
      "avatar_url": null,
      "rating": 4.9,
      "skills": ["Flutter", "Dart", "Firebase"],
      "completed_projects": 23,
      "hourly_rate": 300,
      "tagline": "全栈移动端开发专家"
    }
  ],
  "meta": {
    "page": 1,
    "page_size": 20,
    "total": 8,
    "total_pages": 1
  }
}
```

Mock 文件：`lib/core/mock/mock_data/market_mock.dart`

API 端点常量：`ApiEndpoints.marketExperts` → `/api/v1/market/experts`
